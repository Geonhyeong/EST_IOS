- 현재 진행된 상태

Setting 화면에서 사용자의 ID를 입력하고 서버가 받을 포트(50001)와 사용자가 사용할 포트를 입력하여 Save 버튼을 누른다. 그 후 메인 화면에서 Connect 버튼을 클릭 시 서버와의 연결이 수행되며 등록 커맨드가 전송된다. 서버와 연결이 되면 Connect 버튼위 네모난 회색이 초록색으로 변하며 스피커가 켜지며, talk 버튼이 활성화된다. talk 버튼을 클릭하면 마이크가 켜지게 된다. 연결을 끊을 시에는 talk 버튼 옆 disconnect 버튼을 클릭하면 스피커와 마이크 모두 꺼지며 서버와의 연결도 끊기게 된다. 서버와 연결이 되지 않으면 네모난 회색이 빨갛게 변하며 talk 버튼도 비활성화 된다.

07.20(금) 아이폰 6에 앱을 설치하여 맥북의 가상 디바이스와 아이폰 6간의 서버를 통한 음성 통화 테스트를 시행하였다.
테스트 결과, 음성이 송신과 수신이 되는 것을 확인했으나 약 1초 가량의 지연이 발생하고 노이즈가 섞여 수신 되었다.
또한, 수신되는 음성의 크기가 작아 음량을 최대로 키우고 귀를 기울여야 들리는 정도였다.


- 수정 해야할 사항

1. 현재 Connect 버튼을 누르면 바로 스피커가 켜지는데 이를 talk 버튼 누르면 마이크가 켜지면서 같이 스피커가 켜지도록 수정한다.

2. Setting 화면에서 서버의 포트와 클라이언트의 포트를 적는 부분이 있는데 이를 삭제하고 코드 내에서 지정하는 것으로 수정한다.
-> 클라이언트의 포트를 지정하는 것은 다른 앱과의 충돌이 일어날 수 있으므로 지정된 범위 내에서 랜덤으로 지정한다.

3. 서버로 등록 프로토콜을 보냈을 때 서버에서 다시 recall 해주면 클라이언트에서 받아서 그 후 과정으로 넘어가도록 Callback 함수를 만들어 주어야 한다.

4. 음질을 개선한다.

5. ID를 직접 적는 것보다 소스 내에서 이용자의 주소록에 접근하여 핸드폰 번호를 읽는 방식으로 수정한다.
-> 애플 보안상 불가능 할 수도 있다.

6. 음량 조절 버튼은 테스트 결과 제대로 수행하지 않아 수정해야한다.
